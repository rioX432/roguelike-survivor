# PLAN.md — 8時間ハッカソン 縦画面2Dローグライト

## プロジェクト概要

- **ジャンル**: 縦画面サバイバーズライク（ダダサバイバー系）
- **エンジン**: Unity 6 (URP 2D)
- **ターゲット**: Android / iOS
- **制限時間**: 8時間
- **開発体制**: 人間（ディレクター） + OpenClaw マルチエージェント

---

## MVP スコープ（必達）

| カテゴリ | 内容 |
|---|---|
| プレイヤー | 仮想ジョイスティック移動、自動攻撃 |
| 攻撃 | 3種（放射、前方コーン、追尾弾） |
| 敵 | 5種（通常3 + エリート1 + ボス1） |
| マップ | リングバッファ方式の疑似無限平面 |
| ゲームループ | 10分間の波状スポーン → リザルト画面 |
| UI | HPバー、経験値バー、タイマー、レベルアップ選択 |
| オーディオ | BGM 1曲（ループ）、SE 3〜5種 |
| ビルド | Android APK / iOS (Xcode Archive) |

## ストレッチゴール（余裕があれば）

- 攻撃モジュール追加（設置型、範囲爆発など）
- 敵バリエーション追加
- パッシブスキル選択システム
- スコアランキング
- エフェクト強化（パーティクル）
- BGM 2曲目

## 技術方針

| 項目 | 採用 | 理由 |
|---|---|---|
| エンティティ管理 | GameObject + Object Pool | Jobs Systemは8hでリスク高。パフォーマンス問題が出たら検討 |
| 無限マップ | 3x3チャンク リングバッファ | Tilemap動的生成はカクつくため不採用 |
| データ管理 | ScriptableObject | JSONパース負荷回避、エディタ上で編集可能 |
| UI | Canvas分割（静的/動的） | リビルド負荷軽減 |
| 入力 | InputSystem (New) | マルチプラットフォーム対応 |

## Git競合回避ルール

- シーンファイルは1人だけが編集する（原則builder-alpha）
- builder-betaはPrefabとScriptableObjectのみ編集
- コミットは細かく、機能単位で行う
- `.meta`ファイルは必ずコミットに含める
